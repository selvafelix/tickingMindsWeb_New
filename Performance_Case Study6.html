Case Study: Performance Validation for Leading India Based Air Conditioning Company


Client Background

    Client is a India's largest central air-conditioning company that fulfils the air-conditioning needs of a large number of corporate, commercial and residential customers and has also established leadership in the field of commercial refrigeration.


Business Context

    Client had been using a Call Maintenance System, Preventive Maintenance System, Hand-Over-Form system hosted over the portal and wanted to be prepared for a peak seasonal usage during summer. The management was looking for means to improvise the performance of the systems and we are looking to optimize their spend on the infrastructure, while at the same time looking to achieve performance SLAs.


How did we help the client?

1.  Benchmarked the system by incrementally validating the performance of each of the system one by one, first unearthing application level performance issues, then assessed the system together by simulating the peak anticipated user traffic. Three years anticipated data volume was ported into the database, so as to have realistic measurements.

2.  8-key performance issues were unearthed at application level and once the incremental performance tests were begun, scalability issues were unearthed at just 250 concurrent users. Recommended corrections were implemented by development on database layer like use of clustered indexes, sql query tuning, etc.

3.  Performance bottleneck were identified at the portal server later as well as at the existing database design.


Benefits Delivered

1.  Eliminating performance degrade in the existing setup, recommendations on hardware sizing, helped client to avoid end-user dissatisfaction, which in turn helped the system process more requests efficiently.

2.  Spend optimized by reallocating web server resources to the Application layer and tuning changes done at the server helped provide confidence to business that the system could hold about 450-500 concurrent user traffic and that it could upto 1365 concurrent users without breaking down.
