Case Study: Security Testing for Native Mobile Application


Client Background

 Client is an international humanitarian organization based in the United States. It serves to bring changes in the society and promote equality amongst gender.


Business Context

1.  Client had a text-based, high-priority/high-availability messaging app for use on mobile devices by first-responders and people in similar organizational, community, and societal roles where communication reliability, message security, and minimized delivery latency are of extremely high priority.

2.  The client wanted the application to be security tested, as the system was likely to be used throughout the world by around 10000 users, with lot of messages being confidential, to be contained within the organization.


How did we help the client?

1.  Scanned the iOS, Android and Web application for OWASP vulnerabilities, came with vulnerability findings report and provided recommendations on how to fix those vulnerabilities.

2.  Exploited the system by performing application penetration testing for the platform.

3.  Key tests such as SQLInjection, Cross-Site Scripting, Broken Authentication & Session Management, Insecure Direct Object References, etc. were carried out.


Benefits Delivered

1.  33 critical security defects were unearthed in the process and were later fixed as part of this exercise.

2.   Application vulnerability issues were rectified with shorter turn around based on the precise recommendation provided from our end.

3.  Higher confidence delivered to the Client in their go-live.
